How to Code (Part 1)

Introduction

To start coding, one should have ‘ikigai’ to maintain a happy relationship with your new hobby/career. Instead of finding yourself too broke to live your passion or in a scenario where you suck at something, try and focus on maintaining the good in all areas.

This is ‘Ikigai’ visually:

How to start ?

The key to any skill if it be rugby players or mathematicians is to have a good foundational comprehension.

To achieve this in coding I’d like to suggest to start consuming content of programmers may it be:

  1. Youtube Videos

    • NetworkChuck - Hacking
    • David Bombal - Hacking
    • John Hammond - Hacking
    • Primeagen - Programming

  2. Books

    • My free programming books repository on Github
    • The Self Taught Programmer
    Tip: Google -> "[Book Name] doctype:pdf" <- To find any book you want
  1. The Web
    • Roadmap
    • FreeCodeCamp
    • TheOdinProject
Tip: Use Github and Reddit to find material relevant to what you're busy with

Don’t fuck with memory ! - malloc(Brain.memory);

Start off by using this checklist to make sure you have a clear goal:

  1. Find a programming language that suits your first project

    • Wanna make a website/web app ?
      USE: JAVASCRIPT

    • Wanna make a mobile app ?
      USE: KOTLIN/SWIFT

    • Wanna make a desktop app ?
      USE: C#/C++/Objective C

    • Wanna build Mathematical Models/AI ?
      USE: Python/JS/C++

    • Wanna Hacking ?
      USE: Python/Bash

    The list goes on …

  2. Find GOOD material on the subject matter

    • Preferably start at Object Orientated Programming (OOP)

    • Move on to Roadmap.sh and plan out with ChatGPT

    • Find books on my Github

    • Abuse the internet with Leetcode, HackTheBox, FreeCodeCamp

  3. Start the Hacking, uhhh I mean CODING…

    • You have to sit down and RETYPE, RECOMPILE, RUN, RETHINK, REPEAT
      The five horsemen of learning to code

    • Be CURIOUS about how EVERYTHING works in I.T.

    • Have IDEAS to CODE

    • Show your family and friends and REMEMBER

    • And finally find INSPIRATION

  4. Don’t overuse LLM’s - struggle through it

    • The Primeagen has a great explanation that GOOD programmers use AI’s to HELP them, BAD programmers use AI’s to WRITE for them.
      That doesn’t mean don’t use it AT ALL just don’t ABUSE it.

  5. Start that fucking project bro

    • Do you think Zuck, Zucked when he Zucked in Harvard? NO !
      He was drunk on beer writing the different pages for facebook…

  6. Make your code great again

    • Post that project on Github !
