Casting

So, the first thing everyone should know is that the equal sign operator is a CAST.

x = 5
y = x # Now Y is casted as x and since x is 5, that means y is 5

In the code it is a literal: x is CASTED as a ‘5’. Then we CAST y as x, meaning y is a clone of x.

This may be simple for some of you however this is something I see people struggle with even in high-end I.T. jobs.

Commenting

Comments are made to not run or be understood by the compiler. It’s a way of note-taking if you want to call it that.

# This is a comment :)

If you’re a Developer, this is a reminder to wake up tomorrow morning and write comments in all your previous code. Don’t build legacy code without comments !!! For FUCKsakes…

Variables

You get a variety of variables in different languages but they mostly stay the same.

This table is specifically all the various Data Types in C++.

However in Python we have other Data Types.

x = 4       # x is of type int
x = "Sally" # x is now of type str

x = str(3)    # x will be '3' and be a str
y = int(3)    # y will be 3 and be an int
z = float(3)  # z will be 3.0 and be a float
Example Data Type
x = “Hello” String
x = 20 Integer
x = 20.5 Float
x = 1j Complex
x = [“apple”, “banana”, “cherry”] List
x = (“apple”, “banana”, “cherry”) Tuple
x = range(6) Range
x = {“name” : “John”, “age” : 36} Dictionary
x = {“apple”, “banana”, “cherry”} Set
x = frozenset({“apple”, “banana”, “cherry”}) Frozenset
x = True Boolean
x = b"Hello" Bytes
x = bytearray(5) Bytearray
x = memoryview(bytes(5)) Memoryview
x = None NoneType

Each of these examples have also specific SYNTAX that works on each of them.
